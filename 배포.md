# GitHub Pages 배포 가이드

이 프로젝트는 **React + Vite** 기반으로, `gh-pages` 패키지를 이용해 GitHub Pages에 배포합니다.

---

## 배포 URL

```
https://sudo2100.github.io/products/
```

---

## 사전 준비

### 1. GitHub 저장소 생성
- GitHub에서 `products` 이름의 **public** 저장소를 생성합니다.

### 2. 로컬 저장소와 원격 저장소 연결
```bash
git init
git remote add origin https://github.com/sudo2100/products.git
```

### 3. 의존성 설치 (최초 1회)
```bash
npm install
```

---

## 배포 설정 확인

### `package.json` 설정
```json
{
  "homepage": "https://sudo2100.github.io/products/",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

### `vite.config.js` 설정
```js
export default defineConfig({
  base: '/products/',   // 저장소 이름과 일치해야 합니다
})
```

> **주의:** `base` 값이 GitHub 저장소 이름과 다르면 경로가 깨집니다.

---

## 배포 절차

### 1단계: 변경 사항 빌드 & 배포
```bash
npm run deploy
```
이 명령 하나로 아래 두 작업이 자동으로 실행됩니다.
1. `npm run build` → `dist/` 폴더 생성
2. `gh-pages -d dist` → `dist/` 내용을 `gh-pages` 브랜치에 push

### 2단계: GitHub Pages 브랜치 설정 (최초 1회)
1. GitHub 저장소 → **Settings** → **Pages** 메뉴 진입
2. **Source** → `Deploy from a branch` 선택
3. **Branch** → `gh-pages` / `/ (root)` 선택 후 **Save**

### 3단계: 배포 확인
몇 분 후 아래 주소에서 확인합니다.
```
https://sudo2100.github.io/products/
```

---

## SPA 새로고침 문제 해결

React Router를 사용하는 SPA(Single Page Application)는 GitHub Pages에서 직접 URL을 입력하거나 새로고침하면 404가 발생합니다.  
이 프로젝트는 `vite.config.js`의 `copy-404` 플러그인으로 이 문제를 해결합니다.

빌드 시 `dist/index.html`이 `dist/404.html`로 자동 복사되어, 404 발생 시 앱이 다시 로드됩니다.

---

## 재배포

코드를 수정한 후 다시 배포할 때는 동일하게 실행합니다.
```bash
npm run deploy
```

---

## 배포 중지 방법

### GitHub Pages 비활성화
1. GitHub 저장소 → **Settings** → **Pages** 메뉴 진입
2. **Source** 항목에서 **None** 선택 후 **Save**
3. 배포 URL이 즉시 비활성화됩니다.

### `gh-pages` 브랜치 삭제 (선택)
Pages를 비활성화한 후 브랜치도 삭제하려면 아래 명령어를 실행합니다.
```bash
git push origin --delete gh-pages
```

> **참고:** 소스 코드(`main` 브랜치)는 삭제되지 않으며, 언제든 다시 `npm run deploy`로 재배포할 수 있습니다.

---

## 문제 해결

| 증상 | 해결 방법 |
|------|-----------|
| 배포 후 흰 화면 | `vite.config.js`의 `base` 값이 저장소 이름과 동일한지 확인 |
| 404 오류 (새로고침) | `dist/404.html` 파일이 생성되었는지 확인 |
| `gh-pages` 명령어 오류 | `npm install gh-pages --save-dev` 로 재설치 |
| 변경 사항 반영 안 됨 | 브라우저 캐시 삭제 후 재확인 (GitHub Pages 반영에 수 분 소요) |
